Contributing
============

We welcome contributions! Please see our :doc:`architecture` guide for framework design details.

Code Style
----------

- Follow PEP 8
- Use type hints for all functions
- Write docstrings in Google style
- Run ``make format`` before committing

Testing
-------

- Write unit tests for new features
- Maintain test coverage above 80%
- Run ``make test-unit`` for unit tests or ``make test-all`` for all tests before submitting PR
- Run ``make check`` to ensure code quality (lint + type-check)

Documentation
--------------

- Update docstrings for public APIs
- Add examples for new features
- Update this documentation as needed
